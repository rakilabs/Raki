# R2 — Record the Chunking Design Baseline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a pure `render_chunking_baseline` lib function (whole-vs-chunked deltas + buried-fact winner + honesty header) to `raki-eval`, make `chunk-eval` thin glue over it with a `--write` flag, then run it once to record `docs/eval/chunking-baseline.md` and point the roadmap at P1 as the unblocker.

**Architecture:** The report markdown + winner selection is logic, so it lives in the `raki-eval` library (unit-testable on fabricated `Report`s, no model). The `chunk-eval` binary runs the arms via `run_eval_over`, hands the `Report`s to the renderer, prints, and — under `--write` — writes the synthetic baseline to a git-tracked doc. The real-notes section is unchanged and stays stdout-only (aggregate metrics only — no note text).

**Tech Stack:** Rust, `raki-eval` (`run_eval_over`, `Report`/`MethodScores`/`CategoryReport`), `fastembed` (bge-small + jina-reranker, the real model for the manual run), `std::fs`.

**Spec:** `docs/superpowers/specs/2026-06-10-r2-chunking-baseline-record-design.md` (D1–D6). Prior art: `docs/superpowers/specs/2026-06-06-chunking-eval-substrate-design.md` (substrate + D8 promotion gate).
